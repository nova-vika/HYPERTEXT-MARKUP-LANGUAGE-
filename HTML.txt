------------------------------------------- HOW WEBSITES WORKS ---------------------------------------------


1. ME ( THE USER ):

       ----->>>>  open the crome 
       ----->>>>  type : www.google.com

   >>>>>> I AM ASKING FOR A WEBSITE.

2. BROWER ( CHROME/FIREFOX ):

      ----->>>>  browser understand the request 
      ----->>>>  it says: "i need this website"

3. DNS - DOMAIN NAME SYSTEM ( INTERNET PHONEBOOK ):
       
       ----->>> browser asks DNS : " where is google.com?"
       ------>> DNS replies with a number that is IP address coz computer only know 0's and 1's .

4. SERVER ( WEBSITE COMPUTER ):
       
       ----->>> browser contact the server 
       ----->>> server stores website files 
       ----->>> The server SEND FILES BACK:
                        >>> HTML
                        >>> CSS
                        >>> JavaScript
                        >>> Images

5. BROWSER BUILDS THE WEBSITE:

      ------>>> Browser receives HTML / CSS / JS
      ----->>>> Browser reads HTML first
      ----->>>> Browser applies CSS
      ----->>>> Browser runs JavaScript

6. WEBSITE (What You See):
 
        ------>>> Finally, we see the website on the screen.

--------------------------- AUDIO IN HTML  -------------------------------------------------------------------------

         <audio src="music.mp3" controls></audio>

       >>>  What each part means:

               * <audio> ‚Üí audio player

               *  src ‚Üí audio file path

               *  controls ‚Üí play / pause / volume buttons

                      . Without controls, user sees nothing.

        
     Audio  attributes :

Attribute               	Meaning

autoplay             	Play automatically
loop	                Repeat audio
muted                	Start muted
preload	                Load audio early
